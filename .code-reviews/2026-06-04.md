## Medium

### Dead branch in zero-width unicode severity — always returns `"high"` regardless of bidi
- **File**: `audit/findings.py:338`
- **Issue**: The code reads:
  ```python
  sev: Severity = "high" if bidi_count > 0 else "high"
  ```
  Both branches of the ternary are `"high"`, making the condition a no-op. The comment immediately above says "bidi-override chars are higher severity… Plain zero-width chars are usually a hidden-content attack vector but **lower-severity** than bidi inversion." So plain ZWSP findings should produce a different (lower) severity label, but the bug means they always get `"high"`. The `deduction` on line 339 is correctly differentiated (`3` vs `2`), but the severity label and score are out of sync: a plain ZWSP site deducts 2 points but shows as `"high"`, which is inconsistent with the scoring methodology that defines high as −2 to −3.
- **Fix**: Change line 338 to match the comment's intent:
  ```python
  sev: Severity = "high" if bidi_count > 0 else "medium"
  ```

### Subprocess heuristic is overly broad — matches any `.run()`, `.call()`, `.Popen()` method
- **File**: `audit/ast_scan.py:375`
- **Issue**: The second clause of the subprocess check:
  ```python
  any(name.endswith("." + sp.split(".")[-1]) for sp in _SUBPROCESS_FUNCS if sp.startswith("subprocess"))
  ```
  matches any dotted name ending in `.run`, `.call`, `.check_call`, `.check_output`, `.Popen`, `.getoutput`, or `.getstatusoutput` — regardless of the receiver type. This means `task_runner.run()`, `test_suite.call()`, `asyncio.subprocess.Popen()` (benign), or a `celery.app.run()` call would all be flagged as subprocess invocations. The false positive rate is low for simple MCPs but grows significantly on more complex codebases and will undermine trust in the tool as usage broadens.
- **Fix**: Remove the broad tail-matching heuristic. The exact-match catalog (`name in _SUBPROCESS_FUNCS`) already covers all the canonical subprocess calls. If instance-method coverage is needed (e.g. `proc = subprocess.Popen(...); proc.communicate()`), add the specific dotted names (e.g. `"Popen.communicate"`, `"Popen.wait"`) to `_SUBPROCESS_FUNCS` explicitly rather than using a suffix heuristic:
  ```python
  if name in _SUBPROCESS_FUNCS:
      # existing logic
  ```

- **File**: `tests/fixtures/bad_mcp/server.py` and `tests/test_audit.py`

Dimension 10 is only exercised by isolated unit tests (`TestUnsafeDeserialization`), not by the end-to-end fixture path. A regression in `_check_unsafe_deserialization` would not be caught by the fixture suite.
- **Fix**: Add `pickle.loads(data)` (or `yaml.load(text)`) to `tests/fixtures/bad_mcp/server.py`, then add an assertion to `TestBadFixture`:
  ```python

      dims = [f.dimension for f in report.findings]
      assert "Deserialization" in dims
  ```
